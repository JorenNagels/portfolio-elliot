# CLAUDE.md

Context for AI agents working on this repo. See `README.md` for the human-oriented overview.

## What this is

Single-page static portfolio site for **Elliot Barham** (event host / stage presenter / TV presenter). Originally a 3 MB single HTML file generated by AI; converted to a Next.js project that builds to static HTML/CSS/JS for **AWS S3** hosting.

The site is one page. There is no routing, no backend, no CMS, no auth. Treat it as a brochure.

## Architecture & conventions

- **Next.js 16 App Router** with `output: 'export'` — never add anything that requires a Node runtime (no API routes, no server actions, no `revalidate`, no dynamic params, no middleware).
- **Server components by default.** The only client component is `src/components/FadeIn.tsx` (it needs `IntersectionObserver`). Don't add `'use client'` unless the component genuinely needs browser-only APIs.
- **Tailwind CSS** for styling. Custom tokens defined in `tailwind.config.ts` (`gold`, `gold.light`, `mid`, `muted`, `border-gold-faint`, `font-display`, `font-sans`). Use these instead of hex literals.
- **`next/font/google`** loads Bebas Neue + DM Sans at build time. Don't replace with Google Fonts CDN `<link>` tags.
- **`next-image-export-optimizer`** generates WebP variants at build via the `prebuild` script. Use `<ExportedImage>` (the package's drop-in `next/image`) for **raster images**. Plain `<img>` is correct for SVGs (the brand logos).

## Where things live

| File / directory            | Purpose                                                                                                                                               |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/app/page.tsx`          | Section composition — change this to add / reorder sections                                                                                           |
| `src/app/layout.tsx`        | HTML shell, fonts, `<title>`/metadata                                                                                                                 |
| `src/app/globals.css`       | Tailwind directives + the few things Tailwind can't express: `fade-up`, `scroll-brands` keyframes, carousel mask, brand-img filter, hero-grid overlay |
| `src/components/*.tsx`      | One file per section. Each is self-contained.                                                                                                         |
| `src/components/FadeIn.tsx` | The only client component. Wrap content in `<FadeIn>` to get scroll-triggered fade-up.                                                                |
| `src/lib/data.ts`           | All content data — `services`, `brands`, `clips`, `heroImage`, `portfolioImages`. **Edit copy here, not in components, when possible.**               |
| `public/images/hero/`       | Hero background image (one file)                                                                                                                      |
| `public/images/portfolio/`  | Gallery photos                                                                                                                                        |
| `public/images/brands/`     | Brand SVG logos                                                                                                                                       |
| `next.config.mjs`           | `output: 'export'`, image optimizer env vars                                                                                                          |
| `tailwind.config.ts`        | Color / font theme tokens                                                                                                                             |

## Commands

```bash
npm run dev      # local dev
npm run build    # runs prebuild (optimizer) then next build → out/
```

The `prebuild` hook is set in `package.json` and runs automatically before `build`. There is no separate optimization command.

## Things to avoid

- **Don't introduce features that break static export** — API routes, dynamic `[slug]` without `generateStaticParams`, server actions, `cookies()`, `headers()`, `next/image` with the default loader, `revalidate`, etc.
- **Don't bypass the image optimizer** for new raster images — always use `<ExportedImage>` with a `sizes` prop. SVGs stay as plain `<img>`.
- **Don't add backwards-compat shims, feature flags, or "future-proofing"** abstractions. This site has one consumer (one browser request, one page) — keep it concrete.
- **Don't reach for a CMS, MDX, headless data layer, or i18n**. Content edits go in `src/lib/data.ts`.
- **Don't optimize fonts or images at runtime.** Everything is pre-baked at build.

## Design fidelity

The original `elliot-barham-portfolio (1).html` is preserved in the repo root as the design reference. When in doubt about spacing, color, or animation, open that file alongside the running dev server and compare visually. Section IDs (`#services`, `#credits`, `#clips`, `#testimonial`, `#contact`) match between the two so the nav anchors and JS feel identical.

## Deployment target

Output is uploaded to an **AWS S3 bucket** with static website hosting enabled. `trailingSlash: true` is set so that S3's directory-style URL serving works correctly. If a CDN (CloudFront) is added later, make sure it respects `index.html` for directory requests.

## Adding a new section

1. Create `src/components/NewSection.tsx` as a server component. Use `<FadeIn>` for any elements that should scroll-fade.
2. Add an entry in `src/lib/data.ts` if it's data-driven.
3. Add `<NewSection />` to the `<main>` in `src/app/page.tsx`.
4. Add a nav link to `src/components/Nav.tsx` (`href: '#new-section'`).
